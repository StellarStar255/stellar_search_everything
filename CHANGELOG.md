# Changelog

本项目的所有重要变更都会记录在此文件中。发布新版本时，请在下方添加对应版本的小节，
tag 推送后 CI 会自动提取该小节作为 GitHub Release 的说明。

## [1.6.0] - 2026-07-20

### ✨ 后台驻留

- **关闭主窗口后转入后台**：显示系统托盘图标（macOS 在顶部菜单栏），
  右键菜单提供「显示主窗口 / 检查更新 / 退出」；点 Dock 图标或托盘图标可恢复窗口，
  「退出」才真正退出程序
- 「检查更新」从主界面状态栏移至托盘菜单；后台发现新版本时弹系统通知

### 🐛 修复

- **一键升级报 SSL 证书错误**（CERTIFICATE_VERIFY_FAILED）：打包后的运行环境
  没有系统 CA 证书，现内置 certifi 证书链

## [1.5.0] - 2026-07-20

### ✨ 新功能

- **结果类型筛选**：新增「全部 / 仅文件 / 仅文件夹」选项，默认全部；
  「仅文件」模式仍会递归进入所有子文件夹查找文件，选择会保存

### 🐛 修复

- **改关键词后要按两次回车**：回车之前是「搜索/取消」切换，上一次搜索还在后台
  进行时第一次回车会被当成取消。现在输入框回车永远表示「立即按当前关键词搜索」，
  正在搜索时自动先取消再重开；「搜索」按钮保持原有的切换行为

## [1.4.0] - 2026-07-20

### ✨ 一键升级

- 状态栏新增「检查更新」：启动后自动静默检查 GitHub Releases，发现新版本时
  按钮变为「升级到 vX.Y.Z」，点击即自动下载并安装，完成后自动重启，全程无需手动操作
  - macOS：自动挂载 DMG 替换应用（并清除隔离属性，重启后无需右键打开）
  - Windows：安装向导静默运行
  - Linux：弹出系统授权框后自动安装 deb
- 窗口标题显示当前版本号
- 注意：此功能从本版本起生效，1.3.1 及更早版本仍需手动安装本版本一次

## [1.3.1] - 2026-07-20

### 📦 安装包格式改为各平台原生格式

| 平台 | 安装包 | 安装方式 |
| --- | --- | --- |
| macOS (Apple Silicon) | `.dmg` | 打开后把应用拖入 Applications；首次打开请右键 → 打开 |
| Windows (x86_64) | `-setup.exe` | 双击运行安装向导，自动创建开始菜单项，可正常卸载 |
| Linux (x86_64) | `.deb` | `sudo dpkg -i` 安装，自带应用菜单图标 |

程序功能与 1.3.0 相同。

## [1.3.0] - 2026-07-20

### ✨ 新功能

- **全词匹配**：新增「全词匹配」选项，关键词前后必须是分隔符才算命中——
  搜 `a3` 能找到 `zhiyuan_a3_video`，不再被 `e183a375` 这类哈希串刷屏；
  文件名和内容搜索均支持，中文关键词不受影响，选择会保存

### 🚀 性能

- **大量结果不再卡顿**：结果表格重构为数据模型架构（QTableView + Model）。
  10 万条结果实测：点表头排序 4.7 秒 → 0.02 秒，搜索完成时的卡顿 2.4 秒 → 0.03 秒，
  内存占用也显著下降

### 🐛 修复

- **长文件名显示为「…」**：表格默认按词折行，下划线连写的长文件名整体被挤出
  单行行高，只剩省略号；现改为末尾按字符截断，能显示的部分尽量显示
- 文件名 / 路径列增加悬停提示，可查看完整名称和路径
- 修复点「取消」后，已在队列中的结果批次仍会插入表格的问题

## [1.2.0] - 2026-07-20

### 🐛 修复

- **修复取消搜索后旧结果混入新搜索的问题**：快速「取消 → 重新搜索」时，尚未退出的旧搜索线程
  会带着旧关键词/旧匹配模式继续输出结果（表现为「包含所有关键词」模式下出现不含关键词的条目，
  状态栏误显示「搜索已取消」）。现在每次搜索带有递增代数标记，过期线程的残留结果一律丢弃

### ✨ 改进

- 设置应用级图标，任务栏 / Dock / Cmd+Tab 中正确显示应用图标
- CI 构建流程升级到 Node 24 运行时的最新 Actions

## [1.1.0] - 2026-07-03

### 🚀 界面框架迁移到 Qt（PySide6）

- **全新 Qt 界面**（`FileSearchQt.py`）：解决 Linux 上拖拽缩放卡顿、字体渲染异常等 Tk 固有问题
  - 字体直接走系统 fontconfig，任何环境（含 conda）下中文都能正确显示
  - 拖拽缩放流畅（Qt 双缓冲渲染）
  - 复制文件/文件夹改用 Qt 原生剪贴板，三平台通用，不再依赖 pyobjc/AppleScript
- **中英文界面切换**：状态栏"中文/EN"按钮一键切换（macOS 额外提供系统菜单栏菜单），选择保存到配置
- 缺中文字体时启动弹窗提示安装方法（Tk 版）
- 修复 Linux 字体检测：不再误选点阵字体；修复 macOS Cmd+Tab 图标模糊
- 配置文件与旧版完全兼容

## [1.0.0] - 2026-07-03

Stellar Search Everything 首个正式版本 🎉 —— 一个跨平台的本地文件搜索工具（Tkinter GUI，深色主题）。

### ✨ 功能

- **文件名 / 文件内容两种搜索模式**，多线程搜索不卡界面
- **多关键词搜索**：空格分隔，支持「包含所有关键词」或「包含任一关键词」两种匹配方式
- **历史记录**：双击输入框可快速选择最近搜索过的文件夹和关键词
- **结果列表**：按列排序、右键菜单（打开文件 / 打开所在文件夹 / 复制路径 / 复制文件）
- **界面**：深色主题、可调节字体大小、紧凑显示切换，配置自动保存

### 📦 下载安装

| 平台 | 说明 |
| --- | --- |
| macOS (Apple Silicon) | 解压后将 .app 拖入「应用程序」；首次打开如提示无法验证开发者，请右键点击 → 打开 |
| Windows (x86_64) | 解压后直接运行 exe；SmartScreen 提示时点「仍要运行」 |
| Linux (x86_64) | 解压后运行 `./stellar-search-everything`（需要系统安装 tk） |

各文件的 SHA-256 校验值见 Assets 列表。
